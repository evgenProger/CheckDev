Создание новой вакансии.

В системе нет разделения на роли. Каждый пользователь может быть одновременно и работодателем и кандидатом.

  • Первоначально необходимо зарегистрироваться в системе.

    • Заполнить форму регистрации.
    • После регистрации на email адрес придет уведомление о подтверждении.
  • С введенным ранее email адресом и паролем переходим в форму авторизации.
  • Главное окно программы выглядит так

Для того, чтобы создать новую вакансию нужно

  • выбрать пункт – Добавить вакансию.
  • Заполнить форму создания новой вакансии.
  • Заполнить поля ввода
    • Название – название вакансии. Например Senior Java программист.
    • Компания – имя компании или фирмы, в которую устраивается кандидат.
    • Где находится – где находится рабочее место или удаленно.
    • Уровень заработной платны – диапазон или почасовка.
    • Опыт работы – ожидаемый опыт работы.
    • Описание вакансии.

Создание этапов собеседования.

Любое собеседование состоит из нескольких этапов отбора. Обычно на позицию программиста проводят три собеседования

  1. Общее ознакомительное собеседование. На этом собеседовании собирают общую информацию о кандидате: уровень з/п, опыт работы, отношение к командировкам.
  2. Проверка теоретических знаний. Техническое интервью c кандидатом, где спрашивают теоретические вопросы по программированию.
  3. Решение практических задач. Кандидату предлагается решить одну или несколько задач относящихся к работе. Обычно это тестовое задание или решение алгоритмических задач.


  • Название – название этапа (Общее, Техническое, Тестовое задание).
  • Описание – Что будет на этом этапе.
  • Сообщение об успешно пройденном этапе.
  • Сообщение об отказе, если кандидат не выполнил задание или не проходить условия.

Создание заданий.

Каждый этап собеседования состоит из заданий. Задания имеют различный тип и вид.

Создание нового задания.
Форма создания/редактирования нового задания.
  • Короткое название – короткое название задания.
  • Тип поля ввода – форма ввода для заданий может выглядеть: как числовое поле, календарь, список тестов, алгоритмическая задача.
  • Условие прохождения заданий – критерий проверки по которой определяется верно ли сделано задание или нет. Это поле зависит от типа задания. Например для алгоритмической задачи условия могут быть только выполнено верно или нет. Для календаря можно указывать диапозон.
  • Описание заданий – текст содержит описание выполняемого задания.

Пример отображения задания для кандидата.

  • Описание вакансии. Вид для кандидата.
  • Демонстрация этапа. Вид для кандидата.
  • Демонстрация формы задания. Вид для кандидата.
  • Вид алгоритмического задания.

Анализ результатов кандидатов.

Если кандидат успешно прошел собеседование, ему показывается уведомление.

В это время автору резюме приходит уведомление. Он может посмотреть результаты собеседования.
