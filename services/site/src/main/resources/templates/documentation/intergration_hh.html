Описание сервиса "Hunt4 PRO"

  • Создание новой вакансии.
  • Импорт вакансии из HH.ru
  • Создание этапов собеседования.
  • Создание заданий.
  • Анализ результатов кандидатов.
Помощник HR – сервис автоматизации процесса отбора кандидатов.

Какие задачи решает сервис (Боль клиентов).

Все собеседования, что я проходил, были линейные.

  • Первичное ознакомительное собеседование.
  • Техническое интервью.
  • Тестовое задание.
  • Финальное собеседование с командой и менеджерами.

На каждом этапе кандидата можно отсеять. Например:

  • Не прошел по уровню заработной платы
  • Слабая техническая подготовка
  • Не справился с тестовым заданием

Каждый из этапов занимает от 15 минут до 1 часа времени сотрудника фирмы. Первые два этапа собеседования можно полностью автоматизировать. То есть, сотрудники фирмы будут общаться с кандидатами, которые полностью подходят по общим критериям отбора.

Преимущества сервиса:

  • Отбираем только нужных кандидатов. В любом процесс может появляться ошибки из-за человеческого фактора. Например, кандидат понравился по общению, но совсем не подошел по технической стороне. Из-за симпатии к такому кандидату, HR менеджер может перевести такого кандидата на следующий этап. Это будет отнимать время другого сотрудника. Сервис – это машина без эмоций и субъективной оценки. Если кандидат не проходит по критериям отбора, то ему отказывают.
  • Экономит время сотрудников фирмы, как следствие и деньги фирмы. Обычно, чтобы отобрать одного кандидата надо проверить 20 человек. На каждого кандидата тратиться от часа времени сотрудника. Сервис полностью автоматизирует этот процесс. Если кандидат успешно выполнил задания, то ему приходит уведомление и дальше уже HR согласует финальное собеседование с фирмой.

Как это выглядит.

  • Тестовая площадка - http://hr.job4j.ru
  • Пример использования

Существует два режима.

  • Режим работодателя.
    Создает описание вакансии с указанием поле. Город. Удаленка. Вилка. Короткое название. Добавляем этапы.
    1. Общее собеседование. Составляет последовательные вопросы с полями ввода, либо списками.
    2. Указывает какие вопросы являются критичными. Например уровень ожидаемой з/п выше либо ниже рынка. Кандидат не подходит.
    3. Второй этап. Тестовое задание на время. Прямо на сайте открываем форму для ввода кода с тестами. Кнопка выполнить задание. Код вносят в тестовое поле. Задания даются на алгоритмы. Работодатель выбирает из нашего списка либо загружает свои.
    4. Техническое интервью. Из списка наших тестовых. Работодатель выбирает темы, по которым хочет проверить кандидата. Либо загружает свои вариант дополнительные вопросы. Либо указываем там какие темы хочет добавить.
    5. Каждый день. или определенный период. на почту работодателю приходит сводная таблица по отработанным кандидатам. из которых он уже выбирает лучшего. либо автоматически отправляет уведомление. что кандидат не прошел. по следующим критериям. Это минимизирует время. Например на вакансию джуна приходит 1000 резюме в день. Этот сервис решил бы такие проблемы сразу.
    Ссылку на сервис можно вставить на сайт работодателя. Плюс мы постим и продвигаем его вакансию.
  • Для кандидата - это выглядит как автоматизированное собеседование. Надо только ограничить, чтобы выполнение всех пунктов не занимало больше 2 часов времени.
